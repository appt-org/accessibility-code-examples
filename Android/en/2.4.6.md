# Success Criterion 2.4.6 - Headings and Labels

Make sure you use clear headings: think about this carefully during the design phase. Do not forget to mark headings as heading, you can read how to do that in [success criterion 1.3.1](https://appt.nl/kennisbank/richtlijnen/principe-1/richtlijn-1-3/succescriterium-1-3-1.

On Android, you can link labels to input fields.

<pre>
    <code class="language-markup">
<<a href="https://developer.android.com/reference/android/widget/TextView">TextView</a>
    <a href="https://developer.android.com/reference/android/view/View#attr_android:id">android:id</a>="@+id/label"
    <a href="https://developer.android.com/reference/android/widget/TextView#attr_android:text">android:text</a>="@string/appt_label"
    <a href="https://developer.android.com/guide/topics/ui/accessibility/principles#editable">android:labelFor</a>="@id/field"
/>

<<a href="hier">EditText</a>
    android:id="@+id/field"
    <a href="https://developer.android.com/reference/android/widget/TextView#attr_android:hint">android:hint</a>="@string/appt_hint"
/>        
    </code>
</pre>

```xml
<TextView
    android:id="@+id/label"
    android:text="@string/appt_label"
    android:labelFor="@id/field"
/>

<EditText
    android:id="@+id/field"
    android:hint="@string/appt_hint"
/>
```
